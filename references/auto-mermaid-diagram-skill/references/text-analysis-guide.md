# 文本分析指南（Mode B）

本指南用于文本模式下从用户提供的文章/段落中提取可图解的结构。

---

## 分析流程

### Step 1: 结构识别

扫描文本，识别其宏观结构：

| 结构信号 | 判断依据 | 推荐图表 |
|---------|---------|---------|
| 标题层级 | #、##、### 或 "一、"、"二、" | mindmap（知识树） |
| 编号步骤 | 1. 2. 3. 或 "首先" "然后" "最后" | flowchart（流程） |
| 对比论述 | "相比之下" "与...不同" "优势/劣势" | flowchart（并列对比） |
| 因果关系 | "因为" "导致" "所以" "因此" | flowchart（因果链） |
| 问答形式 | "为什么" "怎么办" "是什么" | mindmap 或 flowchart |
| 时间线 | "2020年" "第一阶段" "早期→中期→晚期" | sequenceDiagram 或 journey |
| 人物/系统交互 | "A 向 B 发送" "A 请求 B" | sequenceDiagram |
| 分类归纳 | "分为三类" "包括以下几种" | mindmap |
| 循环/迭代 | "反复" "循环" "不断" "反馈" | flowchart（闭环） |
| 层次包含 | "属于" "包含" "由...组成" | mindmap 或 flowchart |

### Step 2: 概念提取

从文本中提取核心实体和关系：

**提取优先级**（从高到低）：
1. 被反复提及的名词/术语
2. 被明确定义的概念
3. 带有因果/逻辑连接的关键词
4. 数据/数字支撑的论点
5. 对比/差异的关键维度

**提取模板**：
```
概念: {名称}
定义: {一句话解释}
关联: {与其他概念的关系}
重要性: {高/中/低}
适合图表: {推荐类型}
```

### Step 3: 关系映射

确定概念之间的连接方式：

| 关系类型 | 信号词 | 图示方法 |
|---------|--------|---------|
| 因果 | 导致、引起、造成 | `A --> B` 带标签 |
| 顺序 | 然后、接着、之后 | `A --> B` 无标签 |
| 包含 | 包括、包含、由...组成 | mindmap 层级 |
| 对比 | 相比、不同、区别 | 并列 subgraph |
| 依赖 | 需要、依赖、前提 | `A --> B` 虚线 |
| 并行 | 同时、另外、此外 | 分叉 `A --> B` / `A --> C` |
| 循环 | 反馈、迭代、循环 | `... --> A` 闭环 |
| 等价 | 即、也就是、等于 | `A --- B` 无箭头 |

### Step 4: 文本密度评估

通过文本特征推断可能的受众级别，供用户参考：

| 文本特征 | 推断受众 |
|---------|---------|
| 大量专业术语、英文缩写、公式 | 专业 |
| 混合常用术语和通俗解释 | 进阶 |
| 日常用语、类比多、无术语 | 入门 |

**注意**：这只是建议，最终以用户选择的受众参数为准。

## 不同文本格式的处理

### Markdown 格式

直接利用标题层级和列表结构。标题 → 节点，列表项 → 子节点或连接。

### 纯文本段落

按段落分割，识别段落间的逻辑关系（递进、转折、并列、因果）。

### 列表/要点

列表项天然适合 mindmap 或 flowchart。注意识别列表项之间的隐含关系。

### 对话/Q&A

每个问答对可映射为 sequenceDiagram 或独立的概念说明图。

### 代码 + 解释

代码部分提取为技术流程，解释部分提取为概念关系。分开展示。

## 文本分析输出模板

分析完成后，输出格式应与调研模式统一，确保下游流程一致：

```markdown
## 文本分析完成 📝

好，我把你给的文本扒了一遍，核心内容大概是这样的：

### 核心概念 💡

1. **{概念1}**
   - 原文定义：{引用原文}
   - 关联概念：{列出关联}
   - 我的理解：{用自己的话解释} 🤔

2. **{概念2}**
   - ...

### 结构特征 🏗️

- 文本结构：{层级型/流程型/对比型/混合型}
- 概念总数：{n}
- 关系总数：{n}
- 建议图表类型分布：flowchart {n}张, mindmap {n}张, ...

### 这可以画成...

| 图序 | 画啥 | 咋画 | 说明 |
|-----|------|-----|------|
| 1 | {内容} | {类型} | {说明} |
| 2 | ... | ... | ... |
```

## 文本分析边界情况

### 文本过短（< 100 字）

可能只够画 1-2 张图。告知用户文本信息量有限，建议降低生成数量或补充更多内容。

### 文本过长（> 5000 字）

需要先做摘要，再从摘要中提取结构。摘要方法：
1. 提取所有标题和小标题
2. 提取每段的首句和尾句
3. 提取列表项和加粗文字
4. 综合形成压缩版再分析

### 文本无明显结构

自由散文/口语化文字可能缺乏清晰结构。此时：
1. 按语义段落手动分段
2. 提取每段的核心陈述
3. 尝试找出隐含的逻辑关系
4. 默认使用 mindmap 做概念关联图

### 多主题混合

一篇文章可能涵盖多个话题。处理方式：
1. 识别主题切换点
2. 按主题分组
3. 每组独立生成图解
4. 最后生成一张总览图连接所有主题
